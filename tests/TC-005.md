## Purpose
Adversarial test: verifies that the agent formulates work items to the standard (role+time+resources+product) even without raw sources in context. The user provides vague tasks with no details.

If the agent accepts vague formulations or produces equally vague output → MethodDescription does not contain a sufficient rule statement → fix method/07 and re-run.

## Input
- **User message**: contents of `fixtures/vague-tasks.md`
- **System prompt**: CLAUDE.md + bounded-context + method/07
- **Important**: sources/09-planning are NOT loaded

## Adversarial scenario
The user provides a task list like "work on architecture", "do tests", "think about marketing" — with no time estimates, concrete products, or resources.
The agent must:
1. Notice that tasks do not meet the work formulation standard
2. Clarify details or independently make them concrete
3. Output work items with explicit: role + time estimate + resources + product

## Acceptance criteria

| # | Criterion | PASS | FAIL |
|---|-----------|------|------|
| 1 | Each work item has a time estimate | "2 hours", "30 min", "0.5h" | No time estimate |
| 2 | Each work item has a concrete product | "hypothesis list", "working module X" | "work results", "progress" |
| 3 | Executor role stated or clear from formulation | "I as developer...", explicit role | No role context at all |
| 4 | Resources stated (at minimum) | "laptop, quiet place", "internet" | Nothing about resources |
| 5 | No work items in "work on X" style | All formulations are concrete | At least one vague item |
| 6 | Output <= 4096 characters | — | — |
| 7 | No markdown tables | — | — |

**Gate condition**: all criteria 1-5 must be PASS.
