# Стратегатор — Фаза стратегирования (методы 02–06)

## Шаг 1: Неудовлетворённости (метод 02)

**Цель:** обновить список проблем и возможностей.

Шаги:
1. Прочитать текущий список неудовлетворённостей (если есть)
2. Из текста пользователя выделить новые: проблемы, возможности, эмоции
3. Добавить новые позиции с описанием и эмоциональным контекстом
4. Отметить разрешённые, обновить изменившиеся
5. Проверить: каждая активная неудовлетворённость должна быть связана с проектом

Формат вывода: нумерованный список, каждая позиция = одна строка.

---

## Шаг 2: Проекты и приоритеты (метод 04)

**Цель:** обновить список проектов с явными приоритетами.

Шаги:
1. Просмотреть текущие проекты
2. Добавить новые проекты из неудовлетворённостей и стратегии
3. Для каждого: целевой результат + оценка времени
4. Применить критерии отбора для ранжирования
5. Отметить приоритетные (рекомендуется 2-5)
6. Установить явный порядок среди приоритетных

Правила:
- Рекомендуемое число активных проектов: 2-5
- Если больше 5: предложить сгруппировать или деприоритизировать
- Инвестиционные проекты (50-500ч, 3-6 мес) отличаются от текущих (операционных)

Формат вывода: список с явными ID (P-001, P-002...), приоритет в начале.

---

## Шаг 3: Стратегические гипотезы (метод 03)

**Цель:** обновить стратегические черновики (гипотезы о методах достижения целей).

Шаги:
1. Просмотреть существующие черновики
2. Рассмотреть новые идеи из текста пользователя за неделю
3. Сформулировать гипотезы о методах решения неудовлетворённостей
4. Связать гипотезы с конкретными неудовлетворённостями и проектами
5. Определить приоритетные черновики для дальнейшей проработки

Важно: стратегия = коллекция гипотез, не монолитный документ.
Если нет новых идей: предложить перечитать существующие черновики.

---

## Шаг 4: Задачи и трассировка (методы 05, 06)

**Цель:** обновить задачи по приоритетным проектам; проверить связность цепочки неудовлетворённость → проект → задача.

Шаги задач:
1. Для каждого приоритетного проекта — список конкретных задач на неделю
2. Задачи должны быть конкретными (не "поработать над X")

Шаги трассировки:
1. Каждая активная неудовлетворённость → должна иметь ссылку на проект
2. Каждый приоритетный проект → должен иметь список задач
3. Отметить "сироты" (неудовлетворённости без проекта, проекты без задач)

Критерии отбора проектов (метод 05) применяются при необходимости:
- Личные критерии пользователя: время до результата, стоимость, соответствие ценностям
- Шкала: отлично / хорошо / удовлетворительно / неудовлетворительно
- Пересматриваются раз в квартал или при значительных изменениях в жизни
